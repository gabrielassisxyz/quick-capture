# Phase 1 — UI Design Contract

> Visual and interaction contract for two distinct UI surfaces: (1) a Python TUI for instant thought capture, and (2) a Nexus Inbox view extension for the existing web dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Python TUI) + existing Nexus (Alpine.js 3 + Tailwind v4 + DaisyUI v5) |
| Preset | not applicable — extending existing Nexus design system, no shadcn |
| Component library | Rich + prompt_toolkit (TUI); DaisyUI v5 + custom Nexus classes (web) |
| Icon library | none for TUI; inline SVG for Nexus (following existing pattern) |
| Font TUI | Ghostty default (terminal-inherited) |
| Font Web | Space Grotesk (UI) + JetBrains Mono (technical data) — inherited from Nexus |

**Note:** This project does NOT use shadcn. The Python TUI uses Rich/prompt_toolkit with ANSI styling. The Nexus web extension uses the existing Alpine.js + DaisyUI + Tailwind v4 design system documented in `~/repositories/nexus/docs/design-system.md`. All new Nexus UI must match existing patterns exactly.

## UI Surface 1: Capture TUI (Python — Rich + prompt_toolkit)

### Purpose
Appear instantly on Hyprland hotkey, accept multiline text, save and close in under 5 seconds.

### Visual Design

| Element | Specification |
|---------|---------------|
| Background | Terminal inherits Ghostty theme; Rich Panel border uses `cyan` (maps to Nexus accent `#4fc1ff`) |
| Header | Rich Panel with title "Quick Capture" in `bold cyan`, subtitle in `dim` showing keybinding hint |
| Input prompt | `💭 ` prefix (thought bubble emoji + space) |
| Continuation | `... ` for multiline continuation lines |
| Success message | `[green]✓ Saved[/green]` with truncated ID |
| Cancel message | `[dim]Cancelled.[/dim]` |
| Border style | Rich Panel with `border_style="cyan"`, `padding=(0, 1)` |
| Window size | 900×600 (via Hyprland window rules) |

### Interaction Design

| Interaction | Behavior |
|------------|----------|
| Hotkey ($mainMod+Q) | Spawns floating Ghostty window running `quick-capture` |
| Ctrl+S / Ctrl+Enter | Submit text, save to DB, print success, close terminal |
| Escape | Cancel without saving, print "Cancelled.", close terminal |
| Enter | Newline (multiline mode — NOT submit) |
| Tab | Indent (handled by prompt_toolkit) |

### Rich Style Tokens

| Token | Rich Markup | Usage |
|-------|-------------|-------|
| `accent` | `[cyan]` | Header text, borders, status indicators |
| `success` | `[green]` | Save confirmation |
| `muted` | `[dim]` | Keybinding hints, cancel message |
| `bucket-task` | `[green]` | Task bucket badge |
| `bucket-idea` | `[plum2]` or `[magenta]` | Idea bucket badge (maps to spark #c586c0) |
| `bucket-reference` | `[cyan]` | Reference bucket badge (maps to queued #4ec9b0) |
| `bucket-question` | `[blue]` | Question bucket badge (maps to planning #569cd6) |

### Error States (TUI)

| State | Message |
|-------|---------|
| Empty input submitted | No save, no message — just return to input (prompt_toolkit handles) |
| DB write failure | `[red]✗ Failed to save[/red] — check nexus.db` |
| Terminal won't close | Fallback `os._exit(0)` after 2s timeout |

## UI Surface 2: Nexus Inbox View (Alpine.js + Tailwind + DaisyUI)

### Purpose
Display enriched captures from the database, filterable by bucket, with detail view in slide panel.

### Integration Approach
Extend existing Nexus dashboard — add a new "Captures" view alongside the existing Projects view. Reuse all existing design tokens, CSS variables, component patterns, and Alpine.js store architecture.

│  [bucket filter pills: All · Task · Idea · Reference · Question]  │
│  ───────────────────────────────────────────────────────────────── │

## Spacing Scale

Declares values for Nexus web UI (TUI spacing is terminal-relative, not pixel-based):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, pill gaps |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, card gap |
| lg | 24px | Section padding, panel vertical padding |
| xl | 32px | Layout gaps, header bottom margin |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing, header top padding |

Exceptions (inherited from existing Nexus design system):
- `14px` — Card padding (`p-3.5`), matches existing `.nexus-card` pattern — not reducible without visual inconsistency
- `6px` — Badge horizontal padding (`px-1.5`), standard micro-spacing for pill badges
- `2px` — Badge vertical padding (`py-0.5`), standard micro-spacing for pill badges

## Typography

### Web (Nexus Inbox View)

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 13px | 400 | 1.5 | Space Grotesk |
| Label | 11px | 600 | 1.4 | Space Grotesk (uppercase tracking-wider) |
| Heading | 20px | 700 | 1.2 | Space Grotesk |
| Mono | 11px | 400 | 1.4 | JetBrains Mono |

**Note:** 3 font weights (400/600/700) are inherited from the established Nexus design system. Reducing to 2 weights would collapse the label/heading hierarchy — 600 is necessary to distinguish labels from body, and 700 is necessary for headings. This is an intentional exception justified by visual hierarchy requirements.

### Visual Focal Points

| Surface | Primary Focal Point | Eye Path |
|---------|-------------------|----------|
| Capture TUI | Input prompt area (`💭`) — the blinking cursor draws the eye, framed by the cyan panel border | Panel border → input prompt → subtitle hint |
| Nexus Inbox (list) | Card accent bar (top 1px `--status-color`) — color-coded bars draw the eye first | Accent bar → bucket pill → timestamp → original text |
| Nexus Inbox (detail) | Enriched text section (accent-tinted block with left border) — the colored left border and tinted background create visual emphasis | Left border → enriched text → tags → original text |

### Accessibility

| Element | Requirement |
|---------|-------------|
| Detail panel close button | `aria-label="Close"` on close button (icon-only) |
| Bucket filter pills | `aria-pressed="true/false"` on filter buttons |
| Capture cards | `role="button"` and `tabindex="0"` for keyboard navigation |
| Slide panel | `role="dialog"` and `aria-modal="true"` when open |

## Interaction States

### Capture TUI

| State | Visual | Behavior |
|-------|--------|----------|
| Input active | Prompt `💭 ` with blinking cursor | User types multiline text |
| Submitting | Brief flash `[green]✓ Saved (id: abc12345)` | Terminal exits after ~500ms |
| Cancelling | `[dim]Cancelled.` | Terminal exits immediately |
| Error | `[red]✗ Failed to save — check nexus.db` | Terminal stays open for 3s then exits |

### Nexus Inbox View

| State | Visual | Behavior |
|-------|--------|----------|
| Loading | Existing DaisyUI loading state | Fetch /api/captures |
| Empty (no captures) | Centered: "No captures yet" + hotkey hint | Shows empty state component |
| List (captures exist) | Card grid with bucket pills | Cards show original text + bucket + timestamp |
| Filtered (bucket selected) | Only matching bucket cards shown | Active filter highlighted |
| Capture detail (slide panel open) | Slide panel from right | Shows original + enriched text + tags + wikilinks |
| Enriching | Button text changes to `Enriching…` with spinner | POST /api/captures/:id/enrich |
| Enriched | Enriched section replaces "Not yet enriched" | Shows bucket, enriched text, tags, wikilinks |
| Error | Red text inline | Shows error message near relevant action |
