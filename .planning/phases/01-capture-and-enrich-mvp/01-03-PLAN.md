    **1. Create `src/quick_capture/enrich.py`:**
    Module with three key exports:

    `ENRICHMENT_PROMPT` — Constant string with classification rules (Task/Idea/Reference/Question), enrichment rules (preserve meaning, add context, suggest next steps), and instruction to use wiki-query for context. This is per RESEARCH.md Pattern 4 and ENRI-04 requirement.

    `VALID_BUCKETS = {"Task", "Idea", "Reference", "Question"}`

    `parse_enrichment_output(output: str) -> dict` — Parses JSON from opencode run output. Handles: (a) plain JSON object, (b) ```json ... ``` code fence wrapping, (c) mixed output finding last `{` line. Validates bucket is in VALID_BUCKETS. Raises ValueError on complete parse failure.

    `enrich_capture(capture_id: str, conn=None) -> dict` — Full enrichment pipeline:
    1. Get capture from DB via `get_capture()` (ValueError if not found)
    2. Set status to "enriching" via `update_capture()`
    3. Build prompt = ENRICHMENT_PROMPT + original_text
    4. Run `subprocess.run(["opencode", "run", "--format", "json", prompt], capture_output=True, text=True, timeout=120, shell=False)` — **shell=False is critical** for security (command injection mitigation per threat model T-01-02)
    5. Check returncode; if non-zero: reset status to "unprocessed" via `update_capture()`, raise RuntimeError
    6. Call `parse_enrichment_output(result.stdout)`
    7. Call `save_enrichment(capture_id=capture_id, bucket=enrichment["bucket"], enriched_text=enrichment["enriched_text"], tags=enrichment.get("tags", []), wikilinks=enrichment.get("wikilinks", []))`
    8. Return enrichment dict
    9. On TimeoutExpired: reset status, raise RuntimeError
    10. On any other exception: reset status to "unprocessed", re-raise

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI subprocess → opencode run | Capture text passed as prompt argument (command injection surface) |
| API route → opencode run | Capture text from DB passed to subprocess (command injection surface) |
| LLM output → DB | Untrusted JSON from LLM (schema validation surface) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering, Elevation | enrich.py → subprocess | mitigate | shell=False in all subprocess.run calls; text passed as argument not shell interpolation per Pitfall 3 |
| T-03-02 | Tampering | parse_enrichment_output | mitigate | Validate bucket against VALID_BUCKETS set; reject unknown buckets |
| T-03-03 | Denial | enrich_capture | mitigate | 120-second timeout on subprocess.run; reset status on timeout |
| T-03-04 | Tampering | captures.ts POST /:id/enrich | mitigate | Validate capture exists before enrichment; reset status on any failure |
</threat_model>
